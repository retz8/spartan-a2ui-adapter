# Spartan UI — Label

Official doc: https://spartan.ng/components/label

## How it works

`HlmLabel` is an **attribute directive** applied to a native `<label>` element via the `[hlmLabel]` selector. It uses `hostDirectives` to compose `BrnLabel` (brain layer), forwarding the `id` input. Styling is applied via a `classes()` call in the constructor — no CVA variants or sizes.

## Inputs

| Input | Type | Default | Source |
|---|---|---|---|
| `id` | `string` | `brn-label-${autoincrement}` | brain (`BrnLabel`) |

No additional helm-level inputs. No variants, no sizes.

## Applied classes (always)

```
flex items-center gap-2 text-sm leading-none font-medium select-none
group-data-[disabled=true]:pointer-events-none group-data-[disabled=true]:opacity-50
peer-disabled:cursor-not-allowed peer-disabled:opacity-50
peer-data-[disabled]:cursor-not-allowed peer-data-[disabled]:opacity-50
has-[[disabled]]:cursor-not-allowed has-[[disabled]]:opacity-50
```

The directive is non-configurable — these classes are always applied.

## Notable behaviors

- Applied as an **attribute directive** on a native `<label>` element (not a standalone component), so the rendered element is always `<label>`.
- `id` is auto-generated by the brain layer (using `brn-label-${++nextId}`) if not provided — enables linking with form inputs via `for`.
- Disabled styling cascades from parent group state (`group-data-[disabled=true]`) or peer input state (`peer-disabled`, `peer-data-[disabled]`, `has-[[disabled]]`).
- No outputs. No models. No variants. No sizes.

## A2UI mapping notes

- Since `HlmLabel` is an attribute directive (not a component), the wrapper will render a `<label>` host element and apply `hlmLabel` to it.
- Only meaningful property to expose in the catalog: the **text content** (child text node) and optional `for` attribute.
- No interaction events — label is presentational only.
