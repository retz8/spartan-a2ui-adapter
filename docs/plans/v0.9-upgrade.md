# v0.9 Upgrade Plan

When Google ships v0.9 support in the Angular renderer (`@a2ui/angular`), this adapter will be upgraded to match.

## What Changes

- **`catalogs/spartan/v0.9.0/catalog.json`** — new versioned catalog in v0.9 JSON Schema format (`allOf` composition, `ComponentCommon` base, `unevaluatedProperties: false`)
- **`catalogId`** — changes from `catalogId` field to `$id` field (v0.9 convention)
- **`SPARTAN_CATALOG_ID`** — updated to `https://github.com/jiohin/spartan-a2ui-adapter/blob/main/catalogs/spartan/v0.9.0/catalog.json`
- **Fixture messages** — update to v0.9 message types (`createSurface`, `updateComponents`, `updateDataModel`)
- **TypeScript types** — switch imports from `@a2ui/web_core/types/types` to `@a2ui/web_core/v0_9`

## Versioning

Catalog versioning follows `v{a2ui_spec}.{catalog_revision}`:
- `v0.8.0` → current (A2UI spec v0.8, catalog revision 0)
- `v0.9.0` → first v0.9 catalog
- `v0.9.1` → if we add new components within v0.9

The old `v0.8.x` catalog stays in place for backward compatibility during migration. Clients can advertise both `v0.8.0` and `v0.9.0` in `supportedCatalogIds` per A2UI catalog negotiation protocol.

## Trigger

Track Google's A2UI repo for Angular renderer v0.9 release. No action until then.
